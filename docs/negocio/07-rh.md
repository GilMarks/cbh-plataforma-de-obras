# Modulo RH

## Telas
- `/rh/cadastro` - Cadastrar Funcionario
- `/rh/lista` - Lista de Funcionarios

## Acesso
RH (e Master)

---

## Tela: Cadastrar Funcionario (`/rh/cadastro`)

### Proposito
Formulario completo de admissao de funcionarios.

### Regras de Negocio

1. Apenas `nome` e obrigatorio (trimmed non-empty). Todos os demais campos sao opcionais.
2. `codigoInterno` gerado automaticamente como `FUN-{Date.now()}` (timestamp).
3. `notificacao` e `whatsapp` inicializados como `0`.
4. Apos salvar com sucesso: exibe tela de confirmacao por 1500ms e redireciona para `/rh/lista`.

### Secoes do Formulario

| Secao | Campos |
|---|---|
| Dados Pessoais | Nome (req), Sobrenome, Apelido, Data Nascimento, Sexo, Nacionalidade (default: "Brasileira") |
| Documentos | CPF, RG, PIS |
| Contato | Email, Telefone |
| Endereco | CEP, Rua, Numero, Bairro, Complemento, Cidade, Estado |
| Dados Profissionais | Data Admissao, Ocupacao, Escolaridade, Certificacoes |

### Escolaridade
`Fundamental, Medio, Superior, Pos-graduacao`

### Campos Inicializados Mas Sem Input
- `foto: ''`
- `documentos: []`
- `cei: ''`
- `fornecedor: ''`
- `tiposDocumentos: ''`

## Tela: Lista de Funcionarios (`/rh/lista`)

### Proposito
Visualizacao e exclusao de funcionarios cadastrados.

### Regras de Negocio

1. Exclusao requer `confirm()`. Sem verificacao de integridade referencial.
2. Sem funcionalidade de edicao (apenas visualizacao e exclusao).

### Filtro de Busca
Filtra por `nome`, `sobrenome`, `ocupacao` (case-insensitive) ou `cpf` (substring case-sensitive).

### KPI
Total de Funcionarios cadastrados.

### Colunas da Tabela
Nome (avatar com inicial + nome completo + apelido), Ocupacao, CPF, Telefone, Admissao, Acoes (deletar)
